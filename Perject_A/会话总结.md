# 项目分析会话总结

## 项目概述

这是一款 Unity 挖矿放置/点击升级游戏的反编译脚本集合。玩家在限时挖矿会话中点击/自动挖掘岩石获取矿石，在铁砧冶炼矿条，升级镐子和技能树，收集文物，逐步解锁更稀有的矿种。游戏有 Steam 集成、移动端适配、多语言支持（I2 Localization）和完整的成就系统。

---

## 核心系统分析

### 1. 存档系统

- **三层架构**：`IDataPersistence` 接口 → `GameData` 数据容器 → `DataPersistenceManager` + `FileDataHandler`
- 用 `JsonUtility` 序列化到 `Application.persistentDataPath`
- 每 5 秒自动保存，退出和移动端切后台时强制保存
- 有 `.bak` 备份机制，主存档损坏时自动回滚
- 保存分两步：第一次写主文件，第二次验证后复制为备份

### 2. 对象池系统

- 所有池写在一个 `ObjectPool.cs` 单例里，10 种池共用相同的 `Queue<GameObject>` 模式
- 预设数量在 `Start()` 里批量 `Instantiate`，取用 `Dequeue` + `SetActive(true)`，归还 `Enqueue` + `SetActive(false)`
- 池空时 fallback 到 `Instantiate` 新建，归还时自动扩容
- 初始化差异：光束/等离子球额外设 `position`，电力/炸药/圆形碰撞体额外设 `localScale`
- 光束、等离子球、电力、炸药共用 `projectileParent`，其余各有独立父物体

#### 优化方案：统一管理器

讨论了用 `Dictionary<GameObject, ObjectPool<GameObject>>` 统一管理，以及迁移到 Unity 自带 `UnityEngine.Pool.ObjectPool<T>` 的方案：
- `createFunc`、`actionOnGet`、`actionOnRelease`、`actionOnDestroy` 四个回调
- `defaultCapacity` 是内部集合初始容量（不是预创建数量），需手动预热
- `maxSize` 控制归还时的上限，超出直接 `Destroy`
- `prefabLookup` 字典用于 `Return` 时反查对应池子

### 3. 挖矿系统

#### 岩石生成流程
1. `SetRockScreen.TileWaveAnim()` 启动 14 列瓦片波浪动画
2. 每列弹起时 `TileUpAndDown()` 调用 `SpawnRockCount()` 生成岩石
3. 岩石从对象池取出，先挂到列父物体下获取相对坐标，再脱离到 `originalRockParent`
4. 位置在列范围内随机偏移：x ±77，y ±1.5
5. 没有防重叠机制，靠数量少和 Y 轴排序规避视觉问题

#### 岩石生命周期
```
t=0.00s  对象池取出，OnEnable，计算血量
t=0.22s  SetRandomRocks() 决定矿种，激活子物体
t=0.22s  ScaleRock 弹出动画（0.18秒，随机大小 2.1~2.7）
t=0.40s  isRockSpawnedIn=true，可以被击打
```

#### 矿种决定机制
- 所有矿种概率定义在 `SkillTree` 静态字段
- `SetRandomRocks()` 用递减概率池从最稀有（Painite）往下判断
- 每种矿石先判断 chunk 再判断 full，各自独立随机
- 概率受技能树升级、文物效果影响
- 初始只有金矿（chunk 11%，full 4.2%），其他需技能树解锁

#### 岩石结构
- 所有矿种共用一个 prefab，不同矿石是子物体，通过 `SetActive` 切换显示
- 每种矿石有完整形态、破损形态、chunk 形态三个子物体
- 没有用枚举或数组管理，全部是独立的具名字段（50+个）
- 血量过半时切换为破损贴图，归零时发放奖励并归还对象池

#### 受击动画
- 用协程 `ScaleXY()` 做挤压弹跳：x 缩小到 87%，y 拉伸到 118%，总时长 0.2 秒
- 大岩石弹跳幅度更小（x 94%，y 101%）

### 4. 镐子系统

#### 镐子属性（14把）
- mineTime（挥动间隔）：0.34s → 0.20s
- minePower（伤害）：2.5 → 8.0
- 2XChance（双倍概率）：1.1% → 6.5%
- miningAreaSize（范围）：1.21 → 1.80

#### 镐子生命周期
- 从对象池取出，`OnEnable` 播放 `Animation` 组件的挥动动画（"Swing"/"SwingLeft"）
- 0.25 秒后激活碰撞体，0.04 秒后 `SetActive(false)` 归还对象池
- 总生命周期约 0.31 秒

#### 镐子来源
1. 玩家手动挖（`HandMechanics.Mine()` 协程定时开关碰撞体）
2. 技能树自动生成（`SpawnProjectiles.PickaxeTimer()` 每隔 X 秒）
3. 击中时概率触发额外镐子
4. 等离子球碰撞时概率触发

#### HandMechanics 核心机制
- 碰撞体不是持续开启，而是每隔 `currentMineTime` 开启 0.04 秒再关闭
- 每次关闭再开启触发一次 `OnTriggerEnter2D`，保证单次只生成一个镐子
- 能量饮料天赋激活时间隔减半

### 5. 伤害判定

- 用 `OnTriggerEnter2D` 检测碰撞
- 通过 `gameObject.layer` 区分镐子（6/11/12/13）和锤子（7）
- 通过 `gameObject.tag` 区分闪电束（Beam_PH/Beam_S）、炸药（Dynamite）、等离子球（PlazmaBallCollider）、火焰（Fire）

### 6. 奖励发放

#### 矿石资源动画
- 岩石血量归零时调用 `SpawnMaterialPopUp()`
- 从对象池取出文字物体，用 `localScale` 编码矿种信息（金=0.25，铜=0.35，...Painite=0.95）
- 文字物体独立于岩石，不是子物体
- 弹出动画 0.25s → 停留 0.1~0.6s → 飞向资源框 0.3~0.4s
- 屏幕文字超过 430 个时跳过动画直接加资源（性能保护）

#### 资源 UI 跳动
- `GoldAndOreMechanics.ScaleGoldTextAnim()` 协程
- 放大 10%~40%（0.06~0.12秒）→ 缩回原始大小
- 随机的放大比例和持续时间避免机械感

### 7. 场景切换

- **没有使用 `SceneManager`**，所有内容在同一个 Unity Scene 里
- 靠 `SetActive` 切换 UI 面板模拟场景切换
- 用一组 `static bool` 标志位判断当前场景（`isInUpgrades`、`isInMiningSession` 等）
- 挖矿场景和主菜单之间有圆形遮罩过渡动画（协程 + `Vector3.Lerp`）

### 8. 地图系统

- 196 个六边形瓦片，14 列排列
- 瓦片从对象池取出，每列 14 个
- 波浪动画：14 列依次弹起，每列间隔 0.065 秒，弹起高度 63 单位，持续 0.21 秒
- 渲染排序：`sortingOrder = -(int)y`，Y 越小渲染越靠前

## 架构问题与优化建议

### 问题一：大量 static 字段和 bool 标志位
- 场景状态用多个 `static bool` 管理，可能出现互斥矛盾
- **建议**：用枚举替代 bool 组，或引入状态机
- 对于这个游戏，两种状态（Shop / Mining）就够用

### 问题二：没有事件中心
- 脚本之间全部直接引用调用，耦合度高
- **建议**：引入事件系统或 ScriptableObject 事件频道

### 问题三：代码重复严重
- 对象池 10 种池重复相同代码
- 矿种判断 16 个 if 块几乎一样
- **建议**：数据驱动，用结构体数组 + 循环替代

### 问题四：RockMechanics 职责过多
- 一个脚本承担血量、矿种、显示、碰撞、奖励、动画等十几个职责
- **建议**：拆分为 RockMechanics、OreTypeSelector、RockVisualController、RockRewardHandler、RockDamageReceiver

### 问题五：技能树数值硬编码
- 所有价格和效果值写死在代码里
- **建议**：用 `ScriptableObject` 配置表管理

## 新功能设计讨论

### 关卡地图编辑器
- 用 `LevelConfig` ScriptableObject 存储地图数据（一维数组模拟二维，0=空洞 1=瓦片）
- 创建了 `LevelEditorWindow` 编辑器窗口，支持点击和拖动切换瓦片
- 运行时 `LoadLevel()` 将配置转换为有效坐标列表
- `GetRandomSpawnPos()` 随机取瓦片位置 + 偏移量生成岩石

### 对象池迁移到 Unity ObjectPool
- 使用 `UnityEngine.Pool.ObjectPool<T>`
- 通过 `resetAction` 参数处理各池初始化差异
- 预热需要手动 Get 再 Release
- `prefabLookup` 字典实现归还时自动查找对应池

### AttackArea 脚本
- 简化版 `HandMechanics`，仅保留碰撞体定时开关功能
- 协程无限循环，每隔 `mineTime` 开启碰撞体 `collTime` 秒
